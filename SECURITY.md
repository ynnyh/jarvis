# 安全策略

## 报告漏洞

如果你发现安全漏洞，**请不要公开开 Issue**。请通过以下任一方式私下报告：

- **GitHub Security Advisory**（推荐）：前往仓库的 **Security → Advisories → Report a vulnerability**，提交私密报告。这是最安全的方式，只有仓库维护者能看到。
- **邮件**：发送到仓库主页公示的联系方式，主题标注 `[安全漏洞]`。

报告时请尽量包含：

1. 漏洞类型与影响范围
2. 复现步骤（最小可复现示例最佳）
3. 受影响的版本
4. 你认为可能的修复方向（可选）

## 响应时间

- 收到报告后 **48 小时内**确认收到
- **7 天内**给出初步评估（是否接受、严重程度）
- 修复完成后会公开致谢（除非你希望匿名）

## 支持的版本

仅最新一个发布版本会收到安全修复。请保持更新到最新 Release。

## 本地优先架构说明

Jarvis 是**本地优先**（local-first）的桌面应用，设计上最小化攻击面：

- **密钥不入明文配置**：禅道密码、LLM API Key、Bot Token 等一律存操作系统钥匙链（Windows Credential Manager / macOS Keychain），配置文件里只保留占位符。
- **不向第三方服务器上传数据**：除用户主动配置的禅道 / 帆软 / LLM 服务外，应用不向任何服务器发送数据。
- **本地通信**：MCP 工具、LLM 请求均在本地发起，请求目标完全由用户配置决定。

## 贡献者安全准则

提交 PR 时请遵守：

- 不要在代码、测试、文档中硬编码真实凭据、内网地址或个人隐私信息。使用占位符（`example.com`、RFC 5737 的 `192.0.2.0/24` 等）。
- 涉及敏感配置的改动（凭据存储、网络请求目标）请在 PR 描述里说明。
