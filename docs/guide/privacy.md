# 隐私政策

Jarvis 是**本地优先**的个人助手。一句话：**你的数据存在你自己的机器上，Jarvis 不收集、不上传、无遥测。**

## 数据存哪

全部在本机用户目录 `~/.jarvis/`：

| 路径 | 内容 |
|---|---|
| `config.json` | 明文配置（地址、账号、偏好）——**不含密钥** |
| `conversations/` | LLM 对话历史（本地 JSON） |
| `memory/` | 记忆库（sqlite，LLM 抽取的事实） |
| `logs/` | 运行日志（按天滚动，已脱敏） |
| `custom-pets/` | 自定义宠物素材 |

密钥**不**进上述明文文件，而是存操作系统钥匙链（macOS Keychain / Windows Credential Manager）：禅道密码与 sessionCookie、LLM API Key（含各 profile）、Telegram Bot Token、QQ Bot AppSecret。明文配置里这些字段只留占位符 `********`。

## 数据流向

Jarvis 本身没有服务器、不接收你的任何数据。数据只流向**你自己配置的服务**：

| 目标 | 发送内容 | 触发时机 |
|---|---|---|
| 你配置的 **LLM 服务商**（DeepSeek 等） | 对话内容 + 为完成任务附带的上下文（任务标题/描述、你的输入） | 你发起对话 / agent 工具调用 |
| 你的 **禅道服务器** | 任务查询、工时写回请求 | 使用任务功能 |
| 你的 **帆软 BI 服务器** | 工时明细查询 | 使用工时统计 |
| **Telegram / QQ** 平台 | 你收发的消息 | 启用对应渠道 |
| **Gitee**（`latest.json`） | 仅版本检查请求（无个人数据） | 自动更新检查 |

> LLM / 禅道 / 帆软 / IM 各自的数据处理受其**自身隐私政策**约束。Jarvis 只按你的配置把数据发过去，不在任何第三方留副本。

## 不做什么

- ❌ 无遥测、无埋点、无使用分析
- ❌ 不向 Jarvis 作者或任何第三方上传你的数据
- ❌ 诊断日志只在本地；导出（打包 zip）由你**手动**触发，文件存你指定位置

## 你的控制权

- 明文配置可随时编辑；删 `~/.jarvis/` 即完全重置
- 钥匙链密钥可在系统钥匙链工具里查看 / 删除
- 不配 LLM 即不产生任何对外对话数据
